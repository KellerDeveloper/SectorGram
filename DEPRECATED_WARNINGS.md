# Предупреждения о deprecated пакетах

## Что это значит?

Вы видите предупреждения типа:
```
npm warn deprecated inflight@1.0.6
npm warn deprecated rimraf@3.0.2
npm warn deprecated glob@7.2.3
```

Это **не критичные ошибки** - проект будет работать нормально. Эти предупреждения означают, что некоторые зависимости (или их зависимости) используют устаревшие версии пакетов.

## Почему это происходит?

Эти пакеты являются **транзитивными зависимостями** (зависимостями ваших зависимостей):
- `nodemon` может использовать старые версии `glob` и `rimraf`
- Другие пакеты могут использовать `inflight`

## Что делать?

### Вариант 1: Игнорировать (рекомендуется для начала)

Эти предупреждения **не влияют на работу** проекта. Вы можете продолжать разработку без проблем.

### Вариант 2: Обновить зависимости

Я уже добавил `overrides` в `package.json` для принудительного использования новых версий проблемных пакетов.

Чтобы применить обновления:

```bash
cd backend
rm -rf node_modules package-lock.json
npm install
```

### Вариант 3: Использовать npm audit

Проверьте безопасность зависимостей:

```bash
npm audit
npm audit fix  # автоматическое исправление безопасных проблем
```

## Обновленные версии

В `package.json` добавлены `overrides` для:
- `glob`: обновлен до версии ^10.3.10 (исправлены уязвимости безопасности)
- `rimraf`: обновлен до версии ^5.0.5 (поддерживаемая версия)

## Проверка после обновления

После переустановки зависимостей предупреждения должны исчезнуть или значительно уменьшиться:

```bash
cd backend
npm install 2>&1 | grep -i "deprecated" | wc -l
```

Если число предупреждений уменьшилось - отлично!

## Дополнительная информация

- `inflight` - устаревший модуль, рекомендуется использовать `lru-cache`
- `rimraf` v3 - старая версия, v4+ поддерживается
- `glob` v7 - содержит уязвимости безопасности, v10+ исправлены

Эти пакеты обычно используются внутренними инструментами (например, nodemon для очистки файлов), поэтому их обновление происходит автоматически при обновлении основных пакетов.

## Когда стоит беспокоиться?

- Если видите ошибки (не предупреждения) при установке
- Если проект не запускается
- Если `npm audit` показывает критические уязвимости

В остальных случаях предупреждения можно игнорировать.
