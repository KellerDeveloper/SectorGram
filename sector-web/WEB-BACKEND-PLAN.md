# План реализации возможностей бэкенда в веб-приложении

Документ описывает, какие возможности бэкенда уже используются в вебе и что нужно добавить, чтобы веб покрывал весь API.

---

## 1. Текущее состояние

### Уже реализовано в вебе

| Область | Эндпоинты | Где используется |
|--------|-----------|-------------------|
| **Auth** | `POST /auth/register`, `POST /auth/login` | Страницы Login, Register, AuthContext |
| **Users** | `GET /users/me`, `GET /users/search`, `GET /users/online` | Профиль, NewChat (поиск и онлайн), ChatList |
| **Chats** | `GET /chats`, `POST /chats`, `POST /chats/private`, `GET /chats/:id/messages`, `PUT /chats/messages/:id`, `DELETE /messages/:id`, `POST /chats/:id/read` | ChatList, ChatRoom, NewChat + сокеты для сообщений |

### Не используется в вебе

| Область | Эндпоинты | Назначение на бэке |
|--------|-----------|---------------------|
| Auth | `POST /auth/google` | Вход через Google |
| Users | `PUT /users/me` | Редактирование профиля (name, username, bio, avatar) |
| Users | `POST /users/online-status` | Пакетная проверка онлайн по списку userIds |
| Chats | `GET /chats/:id/online` | Список участников чата, которые сейчас онлайн |
| Notifications | `POST /notifications/register` | Регистрация push-токена (для веба — опционально) |
| Stories | `GET /stories`, `POST /stories`, `GET /stories/user/:userId`, `POST /stories/:id/view`, `DELETE /stories/:id` | Лента историй, создание, просмотры, удаление |
| Media | `POST /media/upload/video-note` | Загрузка видеокружка |
| Events | `GET /events`, `POST /events`, `GET /events/:id`, `POST /events/:id/join`, `POST /events/:id/leave` | События: список, создание, просмотр, участие/выход |

Дополнительно на бэке есть: `GET /health` (мониторинг), раздача статики `GET /uploads/*` — для веба достаточно знать URL загрузок для отображения медиа.

## 2. План реализации по приоритетам

### Фаза 1: Профиль и авторизация

1. **Редактирование профиля (PUT /users/me)**
   - Добавить в `api/users.ts`: `updateMe(payload: { name?, username?, bio?, avatar? })`.
   - Страница/модалка «Профиль» или «Настройки»: форма с полями name, username, bio, загрузка avatar (multipart или base64 — смотреть контракт бэка).
   - В шапке или списке чатов — ссылка/аватар с переходом в профиль.

2. **Вход через Google (POST /auth/google)**
   - Подключить Google Sign-In (например `@react-oauth/google` или ручной OAuth2).
   - В `api/auth.ts`: `loginWithGoogle(idToken: string)` → `POST /auth/google` с телом `{ idToken }`.
   - На странице Login — кнопка «Войти через Google», после успеха — `setToken`, обновление user через `getMe()`.

---

### Фаза 2: Чаты — онлайн в комнате

3. **Онлайн в чате (GET /chats/:id/online)**
   - В `api/chats.ts`: `getChatOnline(chatId: string)` → список пользователей онлайн в этом чате.
   - В `ChatRoom`: показывать блок «В чате сейчас» (аватарки/имена), обновлять по таймеру или при открытии чата/событиях сокета, если бэк отдаёт такие события.

4. **Пакетный онлайн-статус (POST /users/online-status)** — по желанию
   - В `api/users.ts`: `getOnlineStatus(userIds: string[])` для массовой проверки.
   - Использовать в списке участников группового чата или в профилях, чтобы не дергать по одному.

---

### Фаза 3: События (Events)

5. **API для событий**
   - Новый модуль `api/events.ts`:
     - `getEvents()` → `GET /events`
     - `getEvent(id)` → `GET /events/:id`
     - `createEvent(payload)` → `POST /events`
     - `joinEvent(id)` → `POST /events/:id/join`
     - `leaveEvent(id)` → `POST /events/:id/leave`

6. **Страницы и UI**
   - Роут `/events` — список событий (карточки: название, дата, место, участники, кнопки «Участвовать»/«Выйти»).
   - Роут `/events/new` — форма создания события (поля по контракту бэка).
   - Роут `/events/:id` — страница события с деталями и кнопками join/leave.
   - В навигации (шапка/меню) — пункт «События».

---

### Фаза 4: Истории (Stories)

7. **API для историй**
   - Новый модуль `api/stories.ts`:
     - `getStories()` → `GET /stories` (лента)
     - `getUserStories(userId)` → `GET /stories/user/:userId`
     - `createStory(payload)` → `POST /stories` (медиа + описание по контракту)
     - `viewStory(id)` → `POST /stories/:id/view`
     - `deleteStory(id)` → `DELETE /stories/:id`

8. **UI историй**
   - Блок «Истории» на главной (горизонтальная лента аватаров/превью по пользователям).
   - Клик по пользователю — просмотр его историй (слайдер/таймер), вызов `viewStory` при просмотре.
   - Создание своей истории: кнопка «Добавить историю» → выбор файла (фото/видео) → `createStory`.
   - В просмотре своей истории — кнопка «Удалить» → `deleteStory`.

---

### Фаза 5: Медиа и видеокружки

9. **Загрузка видеокружка**
   - В `api/media.ts` (или в `api/chats.ts`): `uploadVideoNote(file: File, duration?: number, size?: number)` → `POST /media/upload/video-note` (multipart).
   - Уточнить по бэку: возвращается ли URL готового файла и как привязывать его к сообщению (отдельный тип сообщения / поле в сообщении).

10. **Отправка видеокружка в чат**
    - В ChatRoom: кнопка «Видеокружок» → выбор файла → `uploadVideoNote` → отправка сообщения с привязкой к медиа (формат по контракту бэка/сокета).

11. **Просмотр загрузок**
    - Сообщения с типом «видеокружок» отображать плеером или ссылкой на `GET /uploads/...` (URL из ответа бэка).

---

### Фаза 6: Уведомления (опционально для веба)

12. **Push в браузере**
    - Если бэк принимает web push: в `api/notifications.ts` — `registerPushToken(token)` → `POST /notifications/register`.
    - Страница настроек: запрос разрешения на уведомления, получение токена (Web Push API), вызов `registerPushToken`.
    - Приоритет ниже, т.к. в описании бэка упор на Expo (мобильные).

---

## 3. Технические шаги (общие)

- **Роутинг:** в `App.tsx` добавить маршруты для профиля, событий, историй (см. выше).
- **Навигация:** общий компонент шапки/меню с ссылками: Чаты, События, Истории, Профиль.
- **Типы:** вынести общие типы (User, Chat, Message, Event, Story и т.д.) в `api/` или `types/`, согласовать с ответами бэка.
- **Ошибки и доступ:** единая обработка 401 (редирект на логин), 403/404 — сообщения в UI.
- **Загрузки:** индикаторы при upload и при создании чата/события/истории.
