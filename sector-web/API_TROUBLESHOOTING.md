# Решение проблем с API

## Ошибка: JSON Parse error: Unexpected character: <

Эта ошибка означает, что сервер вернул HTML вместо JSON. Обычно это происходит когда:

### Причина 1: Сервер не запущен

**Решение:**
1. Убедитесь, что backend сервер запущен:
   ```bash
   cd backend
   npm run dev
   ```

2. Проверьте, что сервер отвечает:
   ```bash
   curl http://localhost:4000/users/online
   ```

### Причина 2: Неправильный адрес сервера

**Проблема:** На реальном устройстве `localhost` указывает на само устройство, а не на ваш компьютер.

**Решение:**

1. **Узнайте IP вашего компьютера:**
   ```bash
   # macOS/Linux
   ifconfig | grep "inet " | grep -v 127.0.0.1
   
   # или
   ipconfig getifaddr en0
   ```

2. **Обновите `mobile/src/api/client.js`:**
   ```javascript
   export const API_BASE_URL = "http://ВАШ_IP:4000";
   // Например: "http://192.168.1.100:4000"
   ```

3. **Убедитесь, что устройство и компьютер в одной Wi-Fi сети**

### Причина 3: Ошибка на сервере

**Проверьте логи сервера** - там должна быть информация об ошибке.

**Решение:**
- Проверьте, что MongoDB подключен
- Проверьте, что все зависимости установлены
- Перезапустите сервер

### Причина 4: CORS ошибка

**Решение:**

Убедитесь, что в `backend/src/server.js` настроен CORS:
```javascript
app.use(cors());
```

### Проверка подключения

Добавьте в начало `EditProfileScreen.js` для отладки:

```javascript
useEffect(() => {
  console.log("API_BASE_URL:", API_BASE_URL);
  console.log("Token:", token ? "есть" : "нет");
}, []);
```

### Быстрая проверка

1. Откройте браузер и перейдите на `http://localhost:4000/users/online`
2. Если видите JSON - сервер работает
3. Если видите ошибку - проблема на сервере

### Для web версии

Если используете web версию (`npm run web`), `localhost` должен работать.

### Для реального устройства

Обязательно измените `API_BASE_URL` на IP вашего компьютера!
