# Telegram-подобные функции

Этот документ описывает реализованные функции, похожие на Telegram.

## ✅ Реализованные функции

### 1. Редактирование сообщений
- **Как использовать**: Долгое нажатие на свое сообщение → выбрать "Редактировать"
- **Backend**: Socket событие `edit_message` или REST `PUT /messages/:id`
- **Frontend**: Показывается индикатор "Редактирование" в поле ввода

### 2. Удаление сообщений
- **Как использовать**: Долгое нажатие на свое сообщение → выбрать "Удалить"
- **Backend**: Socket событие `delete_message` или REST `DELETE /messages/:id`
- **Frontend**: Сообщение удаляется из списка (soft delete)

### 3. Статусы прочтения
- **Как работает**: Автоматически отмечаются как прочитанные при открытии чата
- **Отображение**: 
  - ✓ - отправлено
  - ✓✓ - доставлено/прочитано
- **Backend**: Socket событие `mark_read` или REST `POST /chats/:id/read`

### 4. Индикатор печати (Typing Indicator)
- **Как работает**: Автоматически показывается когда пользователь печатает
- **Отображение**: "Имя пользователя печатает..." под списком сообщений
- **Backend**: Socket события `typing_start` и `typing_stop`
- **Автоматика**: Останавливается через 3 секунды бездействия

### 5. Отправка фото
- **Как использовать**: Нажать кнопку 📎 → выбрать фото из галереи
- **Формат**: Base64 (в будущем можно перейти на загрузку файлов)
- **Ограничения**: Максимум 2MB
- **Backend**: Socket событие `send_message` с полем `media`

### 6. Пересылка сообщений
- **Как использовать**: Долгое нажатие на сообщение → "Переслать" → выбрать чат
- **Особенности**: 
  - Сохраняется информация об исходном отправителе
  - Показывается метка "Переслано от..."
  - Работает с текстом и медиа
- **Backend**: Socket событие `forward_message`

### 7. Улучшенные статусы прочтения
- **Отображение**: 
  - ✓ (серый) - отправлено
  - ✓✓ (серый) - доставлено
  - ✓✓ (синий) - прочитано (как в Telegram)
- **Автоматика**: Автоматически отмечаются при открытии чата

### 8. Улучшенное отображение медиа
- **Фото**: Отображаются с превью
- **Видео**: Показывается thumbnail с кнопкой воспроизведения
- **Файлы**: Показывается иконка, имя файла и размер
- **Поддержка**: photo, video, file, audio типы

### 9. Поиск по сообщениям в чате
- **Как использовать**: Нажать кнопку 🔍 в header чата
- **Особенности**: 
  - Подсветка найденного текста желтым цветом
  - Фильтрация сообщений в реальном времени
  - Работает на web и mobile
- **UI**: Поисковая строка появляется под header

### 10. Ответы на сообщения (Reply)
- **Как использовать**: Долгое нажатие на чужое сообщение
- **Отображение**: Показывается превью сообщения над полем ввода
- **Backend**: Поле `replyTo` в сообщении

### 11. Личные чаты
- **Как использовать**: Поиск пользователя → создать чат
- **Особенности**: 
  - Показывается аватар собеседника
  - Показывается имя собеседника вместо названия чата
  - Онлайн статус

### 12. Редактирование профиля
- **Как использовать**: Профиль → Редактировать
- **Возможности**: 
  - Изменить имя
  - Изменить username
  - Добавить bio
  - Загрузить аватар

### 13. Поиск пользователей
- **Как использовать**: Кнопка поиска в списке чатов
- **Поиск**: По имени, email или username
- **Действие**: Создание личного чата прямо из результатов поиска

### 14. Онлайн статус
- **Отображение**: Зеленая точка рядом с именем пользователя
- **Обновление**: В реальном времени через Socket.io

## 🔄 Двухпанельный интерфейс для Web
- **Web**: Список чатов слева, окно чата справа (как в Telegram Desktop)
- **Mobile**: Полноэкранный режим (как в Telegram Mobile)

## 📝 Примечания

- Все функции работают в реальном времени через Socket.io
- Сообщения хранятся в MongoDB
- Медиа файлы пока хранятся как base64 (можно улучшить в будущем)
- Статусы прочтения обновляются автоматически при открытии чата

### 16. Push-уведомления (как в Telegram)
- **Как работает**:
  - На iOS/Android используется `expo-notifications`
  - При входе в приложение устройство регистрирует Expo Push Token
  - Backend сохраняет токен и при новом сообщении отправляет push всем участникам чата (кроме отправителя)
- **Когда приходят**:
  - Когда приложение **свернуто или закрыто**
  - При активном приложении уведомление обрабатывается через глобальный handler (можно настроить поведение)
- **Ограничения**:
  - В Expo Go push корректно работают только на **реальных устройствах**
  - Для продакшена лучше собрать отдельный билд (EAS)

### 15. Реакции на сообщения
- **Как использовать**: Нажать на кнопку 😊 под сообщением или на существующую реакцию
- **Доступные эмодзи**: 👍, ❤️, 😂, 😮, 😢, 🙏, 🔥, 👏, 🎉, 💯
- **Особенности**: 
  - Показывается количество реакций
  - Повторное нажатие удаляет реакцию
  - Обновления в реальном времени
- **Backend**: Socket события `add_reaction` и `remove_reaction`
